Out of Memory Issues

How do I increase my memory?

For Windows Users:

  • Try edit the pcgen.bat file and assign more memory to pcgen, this means changing the -Xmx option:
           java -jar pcgen.jar -Xmx512m

    This assigns 512Mb of RAM and should be enough memory for pcgen to run with lots of sources loaded.

For Mac Users:

  • First, launch a Terminal window. That gives you a command line. Yeah, I know... a Unix command line on a Mac... it creeped me out the first time. ;-)
  • From the command line, cd to the directory where your PCGen install lives. Then launch java to run PCGen with the memory-adding parameters. It's the same syntax as running it on a PC from the command line. Something like:

           # java -Xms 128M -Xmx 512M -jar pcgen.jar

    (That may not be quite right, but something like that. You might have a machine handy where you can verify the syntax.)

    It has the added benefit that you get to see any error messages which are directed to the console (stdout or stderr), which can be a big help in debugging LST files.

For Unix Users:

  • Refer to Mac instructions

For All Users:

  • Don't load sources that you don't need.
  • In Preferences - Don't auto create equipment, use the provided equipment and custom create equipment as needed. To custom create, right click on the basic equipement and add to it.

Java Issues

What Java do I need?

"PCGen 5.12.x and above needs Java 1.5.x or Java 1.6.0_01 (and above)"

"PCGen 5.10.2 and below needs Java 1.5.x"

"PCGen 5.8.1 and below needs Java 1.4.x"

"PCGen 5.6.1 and below needs Java 1.3.x"

How do I find my Java?

"Not good. Im getting "Bad command or file name"

  • Sounds like you don't have java installed then. Go to http://java.sun.com/ and download and install the latest java 1.5.x JRE. You'll then be able to run PCGen.

For Windows Users:

  • First question is, do you have java installed on your machine. If you don't know, go the command prompt and type:
           java -version

  • You should get some kind of a response like:
           java version "1.5.0_06"
           Java(TM) 2 Runtime Environment, Standard Edition (build 1.5.0_06-b05)
           Java HotSpot(TM) Client VM (build 1.5.0_06-b05, mixed mode, sharing)

For Mac Users:

  • What Java run-time environment (JRE) are you running? (I think I already know the answer, but asking anyway.) From a command line (open a Terminal window), enter:

  • You should get some kind of a response like:

           Java HotSpot(TM) Client VM (build 1.5.0_06-b05, mixed mode, sharing)

For Fedora Users:

  • I can't run pcgen on fedora 7. It crashes just after the options directory selection.
  • PCGen does not run under licgcj (GNU's Java library) because it is incomplete (it is not a complete implementation of Java, especially with respect to Swing (the UI))
  • You need to download a complete Java implementation, such as that from Sun Microsystems (available at http://java.sun.com/)

For Ubuntu Users:

  • The #1 issue for PCGen and Ubuntu is that you need to use the Sun Java implementation, not the default that ships with Ubuntu. We have detailed instructions in one of the closed support requests if you need those (just search for Ubuntu when in the support requests list at PCGen's SF site at SourceForge.net PCGen An RPG Character Generator Tracker).


  • You need to run sun java. You can check this with:

    $ java -version

    If not, you might need to install Sun JRE from the repositories (see the Ubuntu wiki for help)

    If Sun java is installed but not run, you need to update the default java by using:

    $ sudo update-alternatives --config java

    or

    $ sudo update-alternatives --set \java /usr/lib/jvm/java-1.5.0-sun/jre/bin/java

    I think both these command do the same thing (with the first command you get a list of possible java's and are asked to choose).

    Manually moving files and placing symbolic links might get the job done but is probably bad for the long-term health of your system. I think things might break with an update.
