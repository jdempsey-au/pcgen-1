Multiple Priesthoods per Deity

Rule Description: Deities may have more than one "Priesthood", each with their own powers and abilities.

PCGen Version: 5.12.x to Current.

File(s) Covered: feat.lst, and deity.lst

Tags used:

VISIBLE, TYPE, PREFEAT, DOMAINS, PREMULT, PREFEAT, ADD:FEAT

Implementation Discussion

In this section we will discuss only those lst tag elements specific to this rules implementation. For more general information on the associated tags, please visit the LST Tag section of the PCGen Documentation.

Implementing "Priesthoods" for deities requires entries in two lst files. These will be entries for the priesthood feats, in the feat.lst file, where specific powers and abilities unique to the priesthood are granted, and the "Deity" entry, in the deity.lst file, where the preisthood feats are selected and applied.

We will discuss below each of these entries and how they work:

Feat File Entry

Tag Used: VISIBLE:EXPORT

What it does: Hides the feat name from PCGen but displays it on the character sheet.

Tag Used: TYPE:Priesthood

What it does: This sets the TYPE of the associated priesthood feat as "Prisethood".

Tag Used: !PREFEAT:1,TYPE=Priesthood

What it does: Sets a feat requirement of not already having a priesthood feat before granting the associated priesthood feat.

Deity File Entry

Tag Used: DOMAINS:x,x|PREMULT:1,[PREFEAT:1,y],[PREFEAT:1,y]

Variables Used (x):Text (Domain name being granted.)

Variables Used (y):Text (Priesthood feat name required. See priesthood feat list above.)

What it does:

  • This is a comma-delimited list of domains that the deity grants to the members pf it's various priesthoods.
  • The domain name MUST match the domain names listed in the domain list file.
  • The PREMULT tag has been added at the end of the DOMAINS tag to apply restrictions on which priesthoods are allowed to select the domain.
  • Multiple DOMAINS tags may be present for a deity. They will all add to the list of the deity's domains.

Tag Used: PREMULT:1,[PREFEAT:1,x],[PREFEAT:1,y]

What it does:

Tag Used: PREFEAT:1,x

Variables Used (x):Text (Name of priesthood. See Feat File Entry above.)

What it does: Sets a feat requirement for the PREMULT tag as requirements for domains being granted.

Tag Used: ADD:FEAT|Priesthood of the Code Monkey,Priesthood of the Data Monkey,
Priesthood of the Doc Monkey,Priesthood of the Monkey

What it does: Will give the character a choice of one of the four priesthood feats listed.
