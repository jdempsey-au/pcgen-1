Multiple Priesthoods per Deity

Rule Description: Deities may have more than one "Priesthood", each with their own powers and abilities.

PCGen Version: 5.12.x to Current.

File(s) Covered: class.lst, feat.lst, and deity.lst

Tags used:

.MOD, DEFINE, VISIBLE, TYPE, PREVAREQ, BONUS:VAR, DOMAINS, ADD:FEAT

Implementation Discussion

In this section we will discuss only those lst tag elements specific to this rules implementation. For more general information on the associated tags, please visit the LST Tag section of the PCGen Documentation.

Implementing "Priesthoods" for deities requires entries in three lst files. These will be a .MOD of the "Cleric" class in the class.lst file, entries for the specific "Priesthoods" allowed in the feat.lst file, and finally the "deity" entry where the preisthood feats are selected and applied in the deity.lst file.

We will discuss below each of these entries and how they work:

Class File Entry

This entry will ".MOD" the "Cleric" class to allow it to select from from between the priesthoods allowed by the character's deity.

Tag Used: DEFINE:Priesthood|0

What it does:

This tag creates and initializes the "Priesthood" variable which will be used to record the priesthood selected for the character.
