.lst - Skills buying abilities

By Andrew A. Maitland (LegacyKing) and Chris Chandler (Barak) for providing the basis for the text.

File(s) Covered: *_abilities.lst, *_skills.lst, miscinfo.lst, statsandchecks.lst

Tags used:

ABILITYCATEGORY, CATEGORY, PLURAL, EDITABLE, EDITPOOL, FRACTIONALPOOL, POOL, VISIBLE,
DEFINE, TYPE, PREABILITY, PRESKILL, PREVARLT, DESC, BONUS:VAR

This is a lesson to assist users in setting up things such as buying abilities via Skill Ranks or known as Skill Tricks. Each section represents a different file and what tags you'll need. Provided with the included tags is a sample in each section. So if you copy these examples verbatim you'll have a good start. Above is referenced the actual tags for the full capabilities. The TOP header bolded text is the file where these tags need to be placed. Each new line should be considered a TAB in the actual file.

So,

ABILITYCATEGORY:ST
CATEGORY:ST

Becomes:

ABILITYCATEGORY:ST <tab> CATEGORY:ST

Miscinfo.lst file: (Found in the gamemode folder)

ABILITYCATEGORY:ST
CATEGORY:ST
PLURAL:STs
EDITABLE:YES
EDITPOOL:NO
FRACTIONALPOOL:NO
POOL:STPool
VISIBLE:YES

--

ABILITYCATEGORY: This makes the Ability Category. This is what will show in the PCGen user interface, under the FEATs tab.

CATEGORY: This is a 'sub' category. You'll note we leave it the same for this example but you can have the same abilit name as long as it has a different category name. For example: CATEGORY:Ambush could contain 'FooBarSneak' and then you could have CATEGORY:Rogue with 'FooBarSneak'

PLURAL: This has to do with the internationalization. An example is: in_feats which sets the Category key to 'in_feats'. <Tir can you add anything to this?>

EDITABLE: This defines whether a user can modify the abilities in the category. NO would be appropriate for class abilities which aren't chosen by the user.

EDITPOOL: This defines whether a user can modify the pool number or not. Each ability has a 'cost' (Default is 1) if this option is set to 'NO' the user cannot add or subtract from the pool.

FRACTIONALPOOL: This defines whether the pool must be a whole number or can be fractions. Some abilities may have a cost less than '1' like .5 or .25.

POOL: This is a formula to determine the pool number.
Example: TL/2 would grant one point every 2 levels to the character.

VISIBLE: Defines whether this category is visible on the UI or hidden. Usefull for hidden abilities and class abilities that require no interaction from the user.

Statsandchecks.lst file: (Found in the gamemode folder)

(added to the line defining intelligence stat):

DEFINE:STMax|TL/2

--

DEFINE: This tag is used to DEFINE a variable. In this case we are defining 'STMax' to have a value of half the character's total level.

 

Skills.lst file:

ST Skill
TYPE:NoStat
VISIBLE:DISPLAY
BONUS:ABILITYPOOL|ST|SKILL.ST Skill.RANK

--

TYPE: We are defining what 'type' this skill belongs to. You can use any type that you feel appropriate.

VISIBLE: 'DISPLAY' - This skill will only be visible in the GUI of PCGen on not appear on the Character sheet. Other valid choices are: YES - Default (Shows up on both PCGen and the Character sheet); NO - Does not appear either on PCGen or the character sheet; GUI - Same as DISPLAY.

BONUS: Breaking down the tag we are saying to give a bonus to the ABILITYPOOL of the ST category for each skill rank placed in the skill. Note: Each skill rank cost 2 points as a Cross-Class Skill.

Ability.lst file: (showing one ST specifically, adjust PRExxx to be appropriate...)

Note the “!PREABILITY:1,CATEGORY=ST,Ac Ba”... this keeps the ability from showing up for choosing after you have already taken it. This is a temporary workaround as there is currently a bug with the ability object not respecting the default of MULT:NO):

Ac Ba
CATEGORY:ST
TYPE:ST.Movement
VISIBLE:YES
!PREABILITY:1,CATEGORY=ST,Ac Ba
PRESKILL:1,Tumble=12
PREVARLT:STCount,STMax
DEFINE:STCount|0
DESC:<appropriate description>
BONUS:VAR|STCount|1

--

The first line is the Name of the Ability, you can have any name you prefer that makes sense. In ths example we are using a shortened name to simplify the lesson.

CATEGORY: This places the ability named in the 'ST' category. Remember we can use the same name, but you must use a different category. NOTE: This category must have been defined in the miscinfo.lst file to be valid.

TYPE: We have a period '.' delimiter to give the ability more than one 'type'. In this case we have 'ST' as a 'type' and we are placing this Ability under the 'Movement' type.

VISIBLE: Setting the visible to YES (which is default) allows the user to see this on the GUI in PCGen.

!PRExxx: This a PRE-Requisite tag, it determines whether the character qualifies for the Ability. The '!' exclamation mark preceeding the tag tells PCGen that the opposite is to be observed. In our first PRExxx tag, the character cannot have this ability already; CATEGORY=ST,<Skill Name> So I cannot have Ac Ba in the ST category already on my character to qualify.
The PRESKILL says I must have x ranks in the skill. '1' indicates how many skills I must meet, and the Skill=x, sets how many ranks I must have in the skill.
PREVARLT says that I must have less STCount than the STMax. If we remember STMax was defined as half my character level, and as we see further down the line STCount is a new defined tag, which will increase by one each time I grab an ability.

DESC: This is where you type the text you want to appear with your chosen ability. This will be displayed on the character sheet UNLESS the ability is set to NO or DISPLAY. In that case you would skip this tag and may instead use SA: in it's place. That tag is discussed in another lesson.

BONUS: This bonus tag is increasing the 'VAR'iable of STCount by 1 each time it is chosen.

Using this, you merely dump skill points into the ST Skill and then flip to the feats tab and the STs sub-tab and choose the ones you want. Hope that helps.

Remember, miscinfo.lst and statsandchecks.lst are gamemode files. They do not get referenced in the .pcc file directly. There are entries about them in the docs if you want to read up on them. Anyway, to make a long story somewhat shorter, what you want to do is go to the “pcgen/system/gamemodes/35e” directory, open the miscinfo.lst file there and add the information I gave you in the post (right below the Salient Divine Abilities entry would work :p). Then open up the statasandchecks.lst file and add the VAR definition as I described before. To call the ability file it's (as you guessed) “ABILITY:ability.lst” in the .pcc file. Basically pretend that an ability.lst is a feat file as far as what tags you can use in it and you'll be ok (fair warning though, there may be exceptions to that rule of thumb).
