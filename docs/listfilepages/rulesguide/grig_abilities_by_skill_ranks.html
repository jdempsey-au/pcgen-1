Abilities by Skill Ranks

Rule Description: Abilities can be bought by taking ranks in the appropriate skill.

PCGen Version: 5.12.1

Gamemode/Dataset: 35e/RSRD

File(s) Covered: miscinfo.lst, statsandchecks.lst, *_abilities.lst, *_skills.lst

Tags used (Gamemode File):

ABILITYCATEGORY, CATEGORY, PLURAL, EDITABLE, EDITPOOL, FRACTIONALPOOL, POOL, VISIBLE, DEFINE, TYPE (miscinfo), TYPE, VISIBILITY, BONUS:ABILITYPOOL, CATEGORY, TYPE, VISIBLE, PREABILITY, PRESKILL, PREVARLT, DESC, BONUS:VAR

NOTE: In this section we will discuss only those LST tags specific to this implementation, and how they are used to implement the subject rule. For more general information on the associated tags, please visit the LST File Tag Index section of the PCGen Documentation.

Implementation Discussion

Implementing "Abilities by Skill Ranks" requires modifications to two "Gamemode" files and two "Data" files. The gamemode files include the miscinfo.lst file where we will be adding a new category of abilities called the "AbilitybySkills", and the statsandchecks.lst file where we will be modifying the Intelligence stat. The data files files include the skill.lst file where we will be creating a new skill called the "Ability by Skill", and the ability.lst file where we will be creating a new ability to be "purchased", in this case an ability called "Ac Ba".

The mechanics of this rule, as implemented on this page, work this way:

  • When the character is created, the "AbilitybySkillMax" variable is initialized and set to half of the characters total level.
  • Each rank taken in the skill named "Ability by Skill" adds one point to the ability pool for the "AbilitybySkill" category of abilities.
  • "AbilitybySkill" abilities become available under the "AbilitybySkill" sub-tab on the feats tab.
  • The first "AbilitybySkill" ability the character takes defines and initializes the "AbilitybySkillCount" variable.
  • Subsequent "AbilitybySkill" abilities taken by the character increases the value of the variable, "AbilitybySkillCount", by one.
  • Once "AbilitybySkillCount" equals or exceeds "AbilitybySkillMax", "AbilitybySkill" abilities become unavailable until the character goes up a level, thereby increasing the "AbilitybySkillMax" variable.

We will discuss, below, each of the LST entries required, and the tags that make them work, to implement "Abilities by Skill Rank". For assistance in understanding how to read this page you may review the Rules Style Guide.

Miscinfo.lst File

The specific miscinfo.lst file entry required to implement "Abilities by Skill Ranks" consists of the definition of a single ability category called "AbilitybySkill". The tags used to do this are explained below:

Note: The miscinfo.lst file is a gamemode file and as such is not referenced in the *.pcc file. It can be found in the pcgen/system/gamemodes/35e directory. The modifications listed below can be placed right below the Salient Divine Abilities entry.

Tag Used: ABILITYCATEGORY:AbilitybySkill

What it Does: This creates a new ability category called "AbilitybySkill". This is what will show in the PCGen user interface, under the FEATs tab.

Tag Used: CATEGORY:AbilitybySkill

What it Does: This sets the new category's "parent" category as the same as the new category.

Tag Used: PLURAL:AbilitybySkills

What it Does: This establishes the "plural" form of the ability category name. (Used primarily for Internatinalization.)

Tag Used: EDITABLE:YES

What it Does: This establishes that that a user can modify the abilities in the category. NO would be appropriate for class abilities which aren't chosen by the user.

Tag Used: EDITPOOL:NO

What it Does: The "AbilitybySkill" ability pool cannot be modified by the user.

Tag Used: FRACTIONALPOOL:NO

What it Does: Restricts the "AbilitiesbySkill" ability pool to whole numbers only.

Tag Used: POOL:AbilitybySkillPool

What it Does: This establishes the base ability pool for AbilitybySkill equal to the value of the variable "AbilitybySkillPool". As this variable hasn't been initialized, the value is zero with no points being added for each level gained.

Tag Used: VISIBLE:YES

What it Does: Makes this category of ability visible in the PCGen GUI.

You will find the completed AbilitybySkill Ability Category Definition below.

Statsandchecks.lst File

The statsandchecks.lst file entry required in this implementation consists of the modification of an existing entry. The following tag is added to the line defining the intelligence stat:

Note: The statsandchecks.lst file is a gamemode file and as such is not referenced in the *.pcc file. It can be found in the pcgen/system/gamemodes/35e directory.

Tag Used: DEFINE:AbilitybySkillMax|TL/2

What it Does: This tag is used to define the variable "AbilitybySkillMax" to have a value of half the character's total level.

You will find the modified Intellegence Stat below.

Skills.lst File

Only one new entry is required in the skill.lst file. This is the new skill "Ability by Skill". The purpose of this skill is to add bonus points to the pool of available points for the "AbilitybySkill" category of abilities, one for each skill rank taken. The tags used to do this are explained below:

Tag Used: TYPE:NoStat

What it Does: We are defining what 'type' this skill belongs to.

Tag Used: VISIBLE:DISPLAY

What it Does: 'DISPLAY' - This skill will only be visible in the GUI of PCGen on not appear on the Character sheet.

Tag Used: BONUS:ABILITYPOOL|AbilitybySkill|SKILL.Ability by Skill.RANK

What it Does: Breaking down the tag we are saying to give a bonus to the ABILITYPOOL of the "AbilitybySkill" category for each skill rank placed in the skill. Note: Each skill rank cost 2 points as a Cross-Class Skill.

You will find the completed Ability by Skill below.

Ability.lst File

The specific entries in the ability.lst file will consist of one entry per line for each of the AbilitybySkill abilities you wish. The first item on each line will be the name of the particular ability being defined. For the purposes of this section we will be creating an ability called "Ac Ba". The tags used to do this are explained below:

Tag Used: CATEGORY:AbilitybySkill

What it Does: This places the &qut;Ac Ba" ability into the "AbilitybySkill" category. (See miscinfo.lst above.)

Tag Used: TYPE:AbilitybySkill.Movement

What it Does: Sets the TYPE for this ability to "AbilitybySkill" and "Movement".

Tag Used: VISIBLE:YES

What it Does: Allows the user to see this ability within the PCGen GUI. The default value for this tag is "YES" so this tag entry is not strictly necessary.

Tag Used: !PREABILITY:1,CATEGORY=AbilitybySkill,Ac Ba

What it Does: Sets a prerequisite for the "Ac Ba" ability. Specifically, the character must not currently have the "Ac Ba" ability that has been defined as belonging to the "AbilitybySkill" category. An ability called "Ac Ba" of any other category will not disqualify the character from selecting this ability. (The inclusion of this tag is a temporary workaround as there is currently a bug with the ability object not respecting the default of MULT:NO.)

Tag Used: PRESKILL:1,Tumble=12

What it Does: Sets a prerequisite for the "Ac Ba" ability. Specifically, the character must have at least 12 ranks in the Tumble skill in order to select this ability.

Tag Used: PREVARLT:AbilitybySkillCount,AbilitybySkillMax

What it Does: Sets a prerequisite for the "Ac Ba" ability. Specifically, the variable "AbilitybySkillCount" must be less than the variable "AbilitybySkillMax" in order for the ability to be selected.

Tag Used: DEFINE:AbilitybySkillCount|0

What it Does: This tag is used to define the variable "AbilitybySkillCount" and to initialize it to the value zero.

Tag Used: DESC:<appropriate description>

What it Does: This is where you type the text you want to appear with your chosen ability. This will be displayed on the character sheet UNLESS the ability is set to NO or DISPLAY. In that case you would skip this tag and may instead use SA: in it's place. That tag is discussed in another lesson.

Tag Used: BONUS:VAR|AbilitybySkillCount|1

What it Does: This bonus tag increases the value of the variable "AbilitybySkillCount" by 1.

You will find the completed "Ac Ba" Ability below.
