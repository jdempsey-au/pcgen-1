General Naming Standards

Keys are to be in all capital letters and may only contain letters, numbers and underscores ("_").

Each word should be separated by an underscore character for ease in reading.

For purposes of standardization the first section is the same as the SOURCESHORT: in the .pcc, on the second, third or more sections of the Key name is the unique ID of the tag.

For purposes of abbreviation the letter limit on the second and third sections of the Key name is 8 and 7.

EQMODs for different item types (Weapon vs. Armor for example) will be done as separate KEYs

Materials which list different pricing for different item types or sub-types (Light Armor vs. Medium Armor for example) will be done as separate KEYs for each price type.

Tag Name: KEY:x

Variables Used (x): Text (Key Name)

What it does:

This is a unique descriptor for the Equipmods modifier. It allows several modifiers to have the same name, which is useful if several are needed for different equipment types. KEY's must be universally unique. Make sure you don't use anything that's is used in any other source you load. And don't just take out the KEY and use the name. Equipment modifier's will work without a KEY but work much better using the KEY than the name.

What it does:

Creates the key.
