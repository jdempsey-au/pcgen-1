Special Case TYPEs

The LST file tag TYPE has many uses within the PCGen, most commonly it is used to sort objects by type within the display or to identify a group of objects to another object that grants a choice from that type. Additionally there are certain types which have uses beyond sorting the display and identification within the data. Examples of this include types used to identify objects for the output sheets and special case types used by in the code for various purposes. This page will list these types and explain their uses. This page will not detail all TYPEs used in our data (that would be a long list) just the ones more commonly used and/or which have uses outside of the dataset which may not be immediately apparent.

Objects which utilize TYPE tags:

  • Classes
  • Companion Modifiers
  • Deities
  • Equipment
  • Equipment Modifiers
  • Feats and Abilities
  • Languages
  • PCC Files
  • Races
  • Skills
  • Spells
  • Templates
  • Weapon Proficiencies

Class TYPEs

See Class TYPE entry.

In addition to sorting classes in the display class type is used by the gameMode to define certain properties of the class as set by the miscinfo.lst tag CLASSTYPE. These properties include how much (if any) CR is added per level of the class, weather or not the class is considered a monster class and if the class is subject to an XP penalty for multiclassing. See also CRFORMULA, ISMONSTER and XPPENALTY

Companion Modifier TYPEs

See Companion Modifier TYPE entry.

Unlike most other PCGen objects, which are defined in a single line, Companion Modifiers can be composed of several lines. Each companion Modifier line begins with a FOLLOWER or MASTERBONUSRACE tag and not a name or KEY. Companion Modifiers use TYPE as the primary identifier instead.

Another unusual property of TYPE in Companion Modifiers is that Spaces can be used, this is normally not recommended.

Commonly used:

Familiar

Animal Companion

Special Mount

Equip Slot TYPEs

Equip Slot TYPEs are specific TYPEs which are used to control how many of a specific equipment TYPE a character can Equip to any one Equipset. Equipment Slots and their associated TYPEs are defined the equipmentslots.lst gameMode file. Since Slot TYPEs control where an item can be equipped no item should have more than one of these types, an item with more than one Slot TYPE will confuse PCGen and may not be equipped into the expected Slot. A Slot TYPE is not required, an item with no Slot TYPE can still be equipped. There is no limit on how many such items can be equipped.

Special Case Output Sheet TYPEs:

Coin
Gem

Items with the Coin or Gem TYPE are not listed in the Equipment List but are instead listed in a separate block labeled Money.


Armor
Ammunition
Shield
Weapon

These four types trigger special blocks for these Equipment TYPEs, they are mutually exclusive.


OffensiveGear

Used to identify Equipment which can be useful in combat during ones turn but which is not a Weapon or Ammunition (and thus already listed under attacks). This is used in some newer stat block formats. Examples might include a wand of Fireballs, a Thunderstone and a Tanglefoot Bag.


DefensiveGear

Used to identify Equipment which can be useful for defense in combat but which is not an Armor or Shield item (and thus already listed under Armor Class). This is used in some newer stat block formats. Examples might include a Cloak of Displacement and a Ring of Invisibility.

LightSource

Used to identify Equipment which can be used to cast illumination. The item should also contain certain QUALITY tags which define the brightness and duration of the light source.

Equipment Modifier TYPEs

See Equipment Modifier TYPE entry.

Equipment Modifier TYPEs are used to determine what equipment the Equipment Modifiers can be applied to. An Equipment item must have at least one TYPE that an equipment modifier has to be applied.

Special Case:

BaseMaterial

The BaseMaterial TYPE is used to identify Equipment Modifiers which represent materials an item can be made from such as Mithral, Adamantine, Steel and Wood. An equipment item can only have one Equipment Modifier with the BaseMaterial TYPE, if a second one is applied the first is automatically removed.

Feat and Ability TYPEs

See Feat TYPE entry.

Feat and Ability TYPEs fall into three general uses, game rule types, class bonus feat or ability types and types used by the output sheets. Game rule types represent the types the game itself groups the feats or abilities into and are derived from the source material. Class bonus feat or ability types are used to create a group of feats or abilities from which a class is granted a selection.

Output Sheet types:
Types used by the output sheets require more explanation since their purpose is not evident in the data files.


SpecialAttack

SpecialQuality

These two types are used in Abilities of CATEGORY=Special Ability (in the RSRD). Every Ability in this category should have one of these types or the other. Depending on the type the output sheet will list the ability in the Special Attacks block or the Special Qualities block.


Extraordinary

SpellLike

Supernatural

Abilities designated as Extraordinary, SpellLike or Supernatural in the source material do so with an abbreviation at the end of the name, (Ex), (Sp) and (Su). The output sheet can use this type to add the abbreviation to the end of the ability name if it is appropriate for that sheet. Not all sheets will want to display the name in that way and by using a type rather than having the abbreviation be part of the name this makes it optional.


AttackOption

Identifies the Feat or Ability as one which provides an alternate attack or modifies a standard attack. This is used in some newer stat block formats.


Aura

Identifies the Feat or Ability as one which represents an Aura or continuous area of effect ability. This is used in some newer stat block formats.


Immunity

Identifies the Feat or Ability as one which represent a specific immunity. This is used in some newer stat block formats.


ModifyAC

Identifies the Feat or Ability as one which provide a circumstantial AC bonus or other defensive benefit. This is used in some newer stat block formats.


Resistance

Identifies the Feat or Ability as one which represent a specific resistance. This is used in some newer stat block formats.


Weakness

Identifies the Feat or Ability as one which represent a specific weakness. This is used in some newer stat block formats.

PCC TYPEs

See PCC TYPE entry.

TYPE in PCC files is used to set the source display tree in the Sources Tab. Unlike most other places TYPE is used PCC TYPEs can contain spaces. PCC TYPE tags cannot have more than three TYPEs.

Caution: If two PCC files use the same TYPE but in a different order the display in the Sources Tab can potentially be disrupted. Side effects can include sources being displayed more than once. Duplicate TYPEs should be used in the same order in every file.

Weapon Proficiency TYPEs

See Weapon Proficiency TYPE entry.

Proficiency types are used to grant proficiencies and to grant bonuses to weapons of a certain proficiency. TYPEs generally represent broad groups of weapons, such as Simple, Martial and Exotic or a smaller, more specific group of weapons such as Axe, Sword, Crossbow and Thrown used for granting bonuses.

BONUS TYPEs

Appending a TYPE to a BONUS tag is completely unrelated to assigning a TYPE to an object however there do exist certain TYPEs which when used on specific BONUS tags have effects which extend beyond it's standard use in the data. The function of appending a TYPE to a bonus is to control how multiple instances of the same BONUS tag are added together or "stack". For details on this see the Notes on BONUS Types on the BONUS page.
