.lst - Converting Feats to Abilities

By Andrew A. Maitland (LegacyKing).

File(s) Covered: *_.pcc, *_abilities.lst, *_feat.lst, miscinfo.lst

Tags used:

ABILITYCATEGORY, CATEGORY, PLURAL, EDITABLE, EDITPOOL, FRACTIONALPOOL, POOL, VISIBLE,
DEFINE, TYPE, PREABILITY, PRESKILL, PREVARLT, DESC, BONUS:VAR

This is a lesson to assist users in converting. Each section represents a different file and what tags you'll need. Provided with the included tags is a sample in each section. So if you copy these examples verbatim you'll have a good start. Above is referenced the actual tags for the full capabilities. The TOP header bolded text is the file where these tags need to be placed. Each new line should be considered a TAB in the actual file.

So,

ABILITYCATEGORY:My Ability Category
CATEGORY:FEAT

Becomes:

ABILITYCATEGORY:My Ability Category <tab> CATEGORY:FEAT

my.pcc file:

Abilities have a different file type from feats. Use ABILITY: to tell the pcc file where to load ability files

Example:

ABILITY:my_dataset_ability.lst

Miscinfo.lst file: (Found in the gamemode folder)

For Homebrews you'll need to set up any special "abilities" in their own pools.

ABILITYCATEGORY:My Ability Category
CATEGORY:FEAT <required all caps for FEAT if this is a typical feat>
<NOTE:Categories may be different, you can make any category you want AS long as it is defined in the MISCINFO file.>


PLURAL:Class Ability Categories
EDITABLE:YES
EDITPOOL:NO
FRACTIONALPOOL:NO
POOL:0 <Can take a variable formula - See 'Skill Tricks'>
VISIBLE:YES <can also be QUALIFY>

--

ABILITYCATEGORY: This makes the Ability Category. This is what will show in the PCGen user interface, under the FEATs tab.

CATEGORY: This is a 'sub' category. You'll note we leave it the same for this example but you can have the same ability name as long as it has a different category name. For example: CATEGORY:Ambush could contain 'FooBarSneak' and then you could have CATEGORY:Rogue with 'FooBarSneak'

PLURAL: This has to do with the internationalization. An example is: in_feats which sets the Category key to 'in_feats'. It also shows up with the 'Plural' of the name. Ex. Feat would be Feats (Feats would display in the drop down bar).

EDITABLE: This defines whether a user can modify the abilities in the category. NO would be appropriate for class abilities which aren't chosen by the user.

EDITPOOL: This defines whether a user can modify the pool number or not. Each ability has a 'cost' (Default is 1) if this option is set to 'NO' the user cannot add or subtract from the pool. For feats, the default is 'YES'.

FRACTIONALPOOL: This defines whether the pool must be a whole number or can be fractions. Some abilities may have a cost less than '1' like .5 or .25. <YES / NO>

POOL:0 unless you want a formula to determine the pool number.
Example: TL/2 would grant one point every 2 levels to the character.

VISIBLE: Defines whether this category is visible on the UI or hidden. Usefull for hidden abilities and class abilities that require no interaction from the user. <Options are - YES, NO, QUALIFY>

Ability.lst file:

This is where your feat will be placed. You will need to add 'CATEGORY' to your feat to make it an ability.

My Feat
CATEGORY:FEAT <note: ALL CAPS if making an actual FEAT; other abilitiy categories can be whatever name you decide upon>
TYPE:General <Same as standard feats '.' period delimiter>
VISIBLE:YES <DISPLAY, NO, YES are the valid entries; QUALIFY is a FREQ currently>
DESC:<appropriate description>
You may continue to use PREFEAT, but it is recommended that you use PREABILITY. See tag for details.


--

The first line is the Name of the Ability, you can have any name you prefer that makes sense. In ths example we are using a shortened name to simplify the lesson.

CATEGORY: This places the ability named in the 'FEAT' category.
NOTE: Any category, besides 'FEAT', must have been defined in the miscinfo.lst file to be valid.
Another FREQ is to allow ABILITYCATEGORY to be made in the pcc file. This has NOT been implemented yet.

TYPE: We have a period '.' delimiter to give the ability more than one 'type'. In this case we have 'General' as a 'type'.

VISIBLE: Setting the visible to YES (which is default) allows the user to see this on the GUI in PCGen.

DESC: This is where you type the text you want to appear with your chosen ability. This will be displayed on the character sheet UNLESS the ability is set to NO or DISPLAY. In that case you would skip this tag and may instead use 'SA:' tag 5.12 and earlier or 'SAB:' tag (5.14) in it's place. You may also use %1, %2 for variables defined. See DESC tag for further details.

 

Unless you've made a new ABILITYCATEGORY, all feats converted to Abilities will show up on the 'Feats' tab as normal. Great examples of new ABILITY CATEGORIES can be seen in the rsrd/srd for the Fighter, Wizard and Rogue class.

Remember, miscinfo.lst is a gamemode file, to define new ability categories or subcategories you MUST create it there. There are entries about them in the docs if you want to read up on them. To call the ability file it's (as you guessed) “ABILITY:ability.lst” in the .pcc file. Basically pretend that an ability.lst is a feat file as far as what tags you can use in it and you'll be ok (fair warning though, there may be exceptions to that rule of thumb).
